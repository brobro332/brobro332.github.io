## ✅ 코딩 테스트 전 정리용

### 1️⃣ 라이브러리

- Counter: 요소의 갯수를 세는데 용이성과 시간복잡도 측면에서 적합
  - 
  ```python
  from collections import Counter

  nums = [1, 2, 3, 2, 1, 1]
  cnt_nums = Counter(nums)

  print(cnt_nums[1]) # 3 

  print(cnt_nums.most_common()[::-1]) # 3
  ```

- deque: popleft() 메서드가 용이
```python
from collections import deque

nums = [1, 2, 3, 4]
deque_nums = deque(nums)

deque_nums.popleft() # nums == [2, 3, 4]
```

- permutation: 순열 - 순서가 존재 
```python
from itertools import permutation

nums = [1, 2, 3]
temp = permutation(a, 2)

print(list(temp)) # temp == [(1,2),(1,3),(2,1),(2,3),(3,1),(3,2)]
```

- combination: 조합 - 순서가 존재하지 않음 == 중복을 허용하지 않음
```python
from itertools import combinations

nums = [1, 2, 3]
temp = combinations(a, 2)

print(list(temp)) # temp == [(1,2),(1,3),(2,3)]
```

### 2️⃣ 탐색

- DFS: stack 또는 재귀함수 이용
  - stack
    - 1. 문제에서 주어진 graph, root(시작 노드) 입력받기
    - 2. 방문 표시할 visited 리스트와 stack 리스트 초기화
    - 3. 스택에 root 노드 append()
    - 4. while 문 시작(stack에 원소가 하나라도 존재하는 동안 반복)
      - 5. stack.pop()
      - 6. 꺼낸 노드를 아직 방문하지 않았다면 visited에 append()
      - 7. stack.extend(graph[node])
  - 재귀함수
    - 
    ```python
    def dfs_recursive(graph, root, visited=[]):
    
    visited.append(root)
    
    for i in graph[root]:
        if i not in visited:
            dfs_recursive(graph, i, visited)
    
    return visited
    ```

- BFS: deque를 이용
  - 1. 문제에서 주어진 graph, root(시작 노드) 입력받기
  - 2. 방문 표시할 visited 리스트 초기화 
  - 3. while 문 시작(queue에 원소가 하나라도 존재하는 동안 반복)
    - 4. queue에서 한 노드를 꺼내기
    - 5. 꺼낸 노드를 아직 방문하지 않았다면 visited에 append()
    - 6. queue에 인접 노드 append()
