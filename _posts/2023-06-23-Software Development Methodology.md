##### **정의**<br>

소프트웨어 개발 방법론이란, <br>
소프트웨어 개발 전 과정에 지속적으로 적용할 수 있는 방법, 절차, 기법 <br>
소프트웨어를 하나의 생명체로 간주하고 개발 시작부터 끝까지 전 과정을 형상화한 방법론 <br>


##### <br> **종류**
###### 구조적 방법론 : ```Structured Development```
  - 기능에 따라 나누어 개발하고 이를 분할과 정복 접근을 통합하는 방식의 방법론
  - 프로세스 중심의 하향식 방법론
  - 구조적 프로그래밍을 위해 나씨 - 슈나이더만 차트를 사용
    - 논리 기술에 중점
    - 제어 논리 구조로 표현

###### 정보공학 방법론 : ```Information Engineering Development```
  - 정보시스템 개발에 필요한 관리 절차와 작업 기법을 체계화한 방법론 <br>
  - 개발주기를 이용해 대형 프로젝트를 수행하는 체계적인 방법론

###### 객체지향 방법론 : ```Object - Oriented Development```
  - 객체 단위로 시스템을 분석 및 설계하는 방법론
  - 객체, 클래스, 메시지를 사용

###### 컴포넌트 기반 방법론 : ```Component Based Development```
  - 컴포넌트를 조립해서 하나의 프로그램을 작성
  - 새로운 기능을 추가하기 좋음
  - 소프트웨어 재사용이 가능
  - 개발 기간 단축으로 생산성이 향상

###### 애자일 방법론 : ```Agile Development```
  - 절차보다는 사람이 중심
  - 변화에 적응성이 뛰어남
  - 유연하고 신속함
  - 유형
    - XP(eXtreme Programming)
      - 5가지 가치 : 용기, 단순성, 커뮤니케이션, 피드백, 존중
      - 12가지 기본 원리
        - 짝 프로그래밍 : 개발자 둘이서 짝으로 코딩
        - 지속적인 통합(CI) : 매일 여러 번씩 소프트웨어를 통합하고 빌드
        - 메타포어 : 공통적인 이름의 체계를 갖고 공통적인 시스템 서술서를 갖게 되면,<br> 
        개발과 의사소통을 돕는다.
    - 스크럼
      - 팀이 중심이 됨
      - 매일 정해진 시간, 장소에서 짧은 시간의 개발을 하는 팀을 위한 프로젝트 관리 중심 방법론
    - 린
      - 도요타의 린 시스템 품질기법을 적용
      - 낭비 요소 제거

###### 제품 계열 방법론 : ```Product Line Development```
  - 임베디드 작성에 유용
  - 특정 제품에 적용하고 싶은 공통된 기능을 정의하여 개발하는 방법론
