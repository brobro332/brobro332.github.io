### 📌 개요
---
DFS / BFS와 최단 경로 알고리즘 모두 그래프 알고리즘의 한 유형으로 볼 수 있다. <br>
그래프 알고리즘은 굉장히 다양한데, 코딩 테스트에서 출제 비중이 낮은 편이지만, <br>
꼭 제대로 알아야 하는 알고리즘이다. <br>
여기서 다루는 개념들을 바르게 이해할 수만 있다면 다양한 응용문제들도 해결할 수 있게 된다. <br>


그래프 알고리즘은 앞서 배운 내용에 기반한다. <br>
예를 들어 크루스칼 알고리즘은 그리디 알고리즘으로 분류되며, <br>
위상 정렬 알고리즘은 큐 또는 스택 자료구조를 활용해야 구현할 수 있다. <br>


먼저 그래프란 노드와 노드 사이에 연결된 간선의 정보를 가지고 있는 자료구조를 말한다. <br>
예를 들어 '여러 개의 도시가 연결되어 있다'와 같은 내용이 등장하면 그래프 알고리즘을 의심해보면 된다. <br>


더불어 그래프 자료구조 중에서 <a style="color:red;">트리 자료구조</a>는 다양한 알고리즘에서 사용되므로 꼭 기억하자. <br>
트리 자료구조는 부모에서 자식으로 내려오는 계층적인 모델에 속한다. <br>
우선순위 큐를 구현하기 위해 최소 힙이나 최대 힙을 이용할 수 있는데, <br>
최소 힙은 항상 부모 노드가 자식 노드보다 크기가 작은 자료구조로서 트리 자료구조에 속한다고 할 수 있다. <br>

||그래프|트리|
|방향성|방향 그래프 혹은 무방향 그래프|방향그래프|
|순환성|순환 및 비순환|비순환|
|루트 노드 존재 여부|없음|있음|
|노드간 관계성|부모와 자식 관계 없음|부모와 자식 관계 있음|
|모델의 종류|네트워크 모델|계층 모델|

<br>

또한 그래프의 구현 방법은 2가지 방식이 존재한다. <br>

- <b>인접 행렬</b> : 2차원 배열을 사용하는 방식 <br>
노드의 개수가 V, 간선의 개수가 E라고 할 때, <br>
인접 행렬을 이용하는 방식은 O(V^2)만큼의 메모리 공간이 필요하며, <br>
두 노드간 간선의 비용을 O(1)의 시간으로 즉시 알 수 있다. <br>
인접 행렬은 다익스트라 알고리즘에 이용되는 방식이다. <br>

- <b>인접 리스트</b> : 리스트를 사용하는 방식 <br>
노드의 개수가 V, 간선의 개수가 E라고 할 때, <br>
인접 리스트를 이용할 때는 O(E)만큼의 메모리 공간이 필요하며, <br>
두 노드간 간선의 비용을 O(V)만큼의 시간을 소요하여 알 수 있다. <br>
인접 리스트는 플로이드 워셜 알고리즘에 이용되는 방식이다. <br>

노드와 간선의 개수가 모두 많으면 다익스트라 알고리즘, 노드의 개수가 적다면 플로이드 워셜 알고리즘을 이용할 수 있다. <br>

### ✅ 서로소 집합
<a style="color:red;">서로소 집합이란 공통 원소가 없는 두 집합을 의미한다.</a> <br>
<a style="color:magenta;">서로소 집합 자료구조란 서로소 부분 집합들로 나누어진 원소들의 데이터를 처리하기 위한 자료구조라고 할 수 있다.</a> <br>
서로소 집합 자료구조는 union과 find 2개의 연산으로 조작할 수 있다. <br>


union 연산은 2개의 원소가 포함된 집합을 하나의 집합으로 합치는 연산이다. <br>
find 연산은 특정한 원소가 속한 집합이 어떤 집합인지 알려주는 연산이다. <br>


서로소 집합 자료구조는 union-find 자료구조라고 불리기도 한다. <br>
두 집합이 서로소 관계인지를 확인할 수 있다는 말은 <br>
각 집합이 어떤 원소를 공통으로 가지고 있는지를 확인할 수 있다는 말과 같다.<br>

#### 서로소 집합 자료구조
서로소 집합 자료구조는 트리 자료구조를 이용하여 집합을 표현하는데, 알고리즘은 다음과 같다. <br>
    1️⃣ 노드의 개수(V) 크기의 부모 테이블을 모든 원소가 자기 자신을 부모로 가지도록 초기화한다. 
    2️⃣ union(합집합) 연산을 확인하여, 서로 연결된 두 노드 A, B를 확인한다.
      ◾ A와 B의 루트 노드 A', B'를 각각 찾는다.
      ◾ A'를 B'의 부모 노드로 설정한다(B'가 A'를 가리키도록 한다).
    3️⃣ 모든 union(합집합) 연산을 처리할 때까지 1번 과정을 반복한다.

##### 소스코드

```python
def find_parent(parent, x):
    if parent[x] != x:
    	return find_parent(parent, parent[x])
    return x

def union_parent(parent, a, b):
	a = find_parent(parent, a)
    b = find_parent(parent, b)
    if a<b:
    	parent[b] = a
    else:
    	parent[a] = b
        
v,e = map(int, input().split())
parent = [0]*(v+1)

for i in range(1, v+1):
	parent[i] = i
    
for i in range(e):
	a,b = map(int, input().split())
    union_parent(parent, a, b)
    
print('각 원소가 속한 집합: ', end='')
for i in range(1, v+1):
	print(find_parent(parent, i), end=' ')
    
print()

print('부모 테이블: ', end='')
for i in range(1, v+1):
	print(parent[i], end=' ')
```

##### 코드 해설
이렇게 구현하면 답은 구할 수 있지만, find 함수가 비효율적으로 동작한다. <br>
최악의 경우 find 함수가 모든 노드를 다 확인하는 터라 시간 복잡도가 O(V)라는 점이다. <br>
결과적으로는 현재의 알고리즘을 그대로 이용하게 되면 노드의 개수가 V개이고 <br>
find 혹은 union 연산의 개수가 M개일 때, 전체 시간 복잡도는 O(VM)이 되어 비효율적이다. <br>


이러한 find 함수는 경로 압축 기법을 적용하면 시간 복잡도를 개선할 수 있다. <br>
경로 압축은 find 함수를 재귀적으로 호출한 뒤에 부모 테이블값을 갱신하는 기법이다. <br>

```python
# 개선된 서로소 집합 알고리즘 소스코드

# 경로 압축 기법을 적용한 find 함수
def find_parent(parent, x):
    if parent[x] != x:
    	parent[x] = find_parent(parent, parent[x])
    return parent[x]

def union_parent(parent, a, b):
	a = find_parent(parent, a)
    b = find_parent(parent, b)
    if a<b:
    	parent[b] = a
    else:
    	parent[a] = b
        
v,e = map(int, input().split())
parent = [0]*(v+1) 

for i in range(1, v+1):
	parent[i] = i

for i in range(e):
	a,b = map(int, input().split())
    union_parent(parent, a, b)

print('각 원소가 속한 집합: ', end='')
for i in range(1, v+1):
	print(find_parent(parent, i), end=' ')
    
print()

print('부모 테이블: ', end='')
for i in range(1, v+1):
	print(parent[i], end=' ')
```

##### 시간 복잡도
union 및 find 함수가 총 100만 번 수행된다고 하면, <br>
약 1,000만 번 가량의 연산이 필요하다고 이해하면 된다. <br>
경로 압축은 개념 및 구현이 간단하다는 점에서 꼭 기억해두어야 한다. <br>

##### 서로소 집합을 활용한 사이클 판별
서로소 집합은 다양한 알고리즘에 사용될 수 있다. <br>
특히 무방향 그래프 내에서의 사이클을 판별할 때 사용할 수 있다. <br>
참고로 방향 그래프의 사이클 여부는 DFS를 이용해 판별할 수 있다. <br>


union 연산을 통해 간선을 하나씩 확인하면서 <br>
두 노드가 포함되어 있는 집합을 합치는 과정을 반복하는 것만으로도 사이클을 판별할 수 있다. <br>

    1️⃣ 각 간선을 확인하며 두 노드의 루트 노드를 확인한다.
      ◾ 루트 노드가 서로 다르다면 두 노드에 대하여 union 연산을 수행한다.
      ◾ 루트 노드가 서로 같다면 사이클이 발생한 것이다.
    2️⃣ 그래프에 포함되어 있는 모든 간선에 대하여 1번 과정을 반복한다.

###### 소스코드

```python
# 서로소 집합을 활용한 사이클 판별 소스코드
def find_parent(parent, x):
    if parent[x] != x:
    	parent[x] = find_parent(parent, parent[x])
    return parent[x]

def union_parent(parent, a, b):
	a = find_parent(parent, a)
    b = find_parent(parent, b)
    if a<b:
    	parent[b] = a
    else:
    	parent[a] = b
        
v, e = map(int, input().split())
parent = [0]*(v+1)

for i in range(1, v+1):
	parent[i] = i

cycle = False

for i in range(e):
	a,b = map(int, input().split())
    if find_parent(parent, a) == find_parent(parent, b):
    	cycle = True
        break
    else:
    	union_parent(parent, a, b)
        
if cycle:
	print("사이클이 발생했습니다.")
else:
	print("사이클이 발생하지 않았습니다.")
```

### ✅ 신장 트리
신장 트리는 그래프 알고리즘 문제로 자주 출제되는 문제 유형이다. <br>
<a style="color:red;">기본적으로 신장 트리란 하나의 그래프가 있을 때 모든 노드를 포함하면서 사이클이 존재하지 않는 부분 그래프를 의미한다.</a> <br>
이 때 모든 노드가 포함되어 서로 연결되면서 사이클이 존재하지 않는다는 조건은 트리의 성립 조건이기도 하다. <br>

#### 크루스칼 알고리즘
우리는 다양한 문제 상황에서 가능한 한 최소한의 비용으로 신장 트리를 찾아야 할 때가 있다. <br>
예를 들어 N개의 도시가 존재하는 상황에서 <br> 
두 도시 사이에 도로를 놓아 전체 도시가 서로 연결될 수 있게 도로를 설치하는 경우를 생각해보자. <br>
모든 도시를 연결할 때, 최소한의 비용으로 모든 도시를 연결하려면 최소 신장 트리를 이용해야 한다. <br>
대표적인 최소 신장 트리 알고리즘으로는 <b>크루스칼 알고리즘</b>이 있다. <br>
최소 신장 트리는 간선의 개수가 '노드의 개수 - 1'과 같다는 특징이 있다. <br>
구체적은 알고리즘은 이러하다. <br>

    1️⃣ 간선 데이터를 비용에 따라 오름차순으로 정렬한다.
    2️⃣ 간선을 하나씩 확인하며 현재의 간선이 사이클을 발생시키는지 확인한다.
      ◾ 사이클이 발생하지 않는 경우 최소 신장 트리에 포함시킨다.
      ◾ 사이클이 발생하는 경우 최소 신장 트리에 포함시키지 않는다.
    3️⃣ 모든 간선에 대하여 2번의 과정 반복한다.

##### 소스코드
```python
# 특정 원소가 속한 집합을 찾기
def find_parent(parent, x):
	# 루트 노드를 찾을 때까지 재귀 호출
    if parent[x] != x:
    	parent[x] = find_parent(parent, parent[x])
    return parent[x]
    
# 두 원소가 속한 집합을 합치기
def union_parent(parent, a, b):
	a = find_parent(parent, a)
    b = find_parent(parent, b)
    if a<b:
    	parent[b] = a
    else:
    	parent[a] = b

# 노드의 개수와 간선(Union 연산)의 개수 입력 받기
v,e = map(int, input().split())
parent = [0]*(v+1)  # 부모 테이블 초기화하기

# 모든 간선을 담을 리스트와, 최종 비용을 담을 변수
edges = []
result = 0

# 부모 테이블상에서, 부모를 자기 자신으로 초기화
for i in range(1, v+1):
	parent[i] = i
    
# 모든 간선에 대한 정보를 입력 받기
for _ in range(e):
	a,b,cost = map(int, input().split())
    # 비용순으로 정렬하기 위해서 튜플의 첫 번째 원소를 비용으로 설정
    edges.append((cost, a, b))

# 간선을 비용순으로 정렬
edges.sort()

# 간선을 하나씩 확인하며
for edge in edges:
	cost, a, b = edge
    # 사이클이 발생하지 않는 경우에만 집합에 포함
    if find_parent(parent, a) != find_parent(parent, b):
    	union_parent(parent, a, b)
        result += cost

print(result)
```

##### 시간 복잡도
크루스칼 알고리즘에서 시간이 가장 오래 걸리는 부분이 간선을 정렬하는 작업이며, <br>
간선의 개수가 E개일 때, O(ElogE)의 시간 복잡도를 가진다. <br>
크루스칼 내부에서 사용되는 서로소 집합 알고리즘의 시간 복잡도는 정렬 알고리즘의 시간 복잡도보다 작으므로 무시한다. <br>

### ✅ 위상 정렬
위상 정렬은 정렬 알고리즘의 일정이다. <br>
<a style="color:red;">위상 정렬이란 방향 그래프의 모든 노드를 방향성에 거스르지 않도록 순서대로 나열하는 것이다.</a> <br>
즉 순서가 정해져 있는 일련의 작업을 차례대로 수행해야 할 때 사용할 수 있는 알고리즘이다. <br>
현실 세계에서 위상 정렬을 수행하게 되는 전형적인 예시로는 '선수과목을 고려한 학습 순서 설정'을 들 수 있다. <br>


위상 정렬 알고리즘을 자세히 살펴보기 전에, 먼저 진입차수를 알아야 한다. <br>
진입차수란 특정한 노드로 들어오는 간선의 개수를 의미한다. <br>
예를 들어 '고급 알고리즘' 노드가 2개의 선수과목을 가지고 있다면, 그래프 상에서 진입차수가 2인 것이라고 말할 수 있다. <br>
위상 정렬 알고리즘은 다음과 같다.

    1️⃣ 진입 차수가 0인 노드를 큐에 넣는다.
    2️⃣ 큐가 빌 때까지 다음의 과정을 반복한다.
      ◾ 큐에서 원소를 꺼내 해당 노드에서 출발하는 간선을 그래프에서 제거한다.
      ◾ 새롭게 진입차수가 0이 된 노드를 큐에 넣는다.

이 때 모든 원소를 방문하기 전에 큐가 빈다면 사이클이 존재한다고 판단할 수 있다. <br>
다시 말해 큐에서 원소가 V번 추출되기 전에 큐가 비어버리면 사이클이 발생한 것이다. <br>
다만, 기본적으로는 위상 정렬 문제에서는 사이클이 발생하지 않는다고 명시하는 경우가 많다. <br>

#### 소스코드
```python
from collections import deque

# 노드의 개수와 간선의 개수를 입력받기
v, e = map(int, input().split())
# 모든 노드에 대한 진입차수는 0으로 초기화
indegree = [0] * (v + 1)
# 각 노드에 연결된 간선 정보를 담기 위한 연결 리스트(그래프) 초기화
graph = [[] for _ in range(v+1)]

# 방향 그래프의 모든 간선 정보를 입력받기
for _ in range(e):
    a, b = map(int, input().split())
    graph[a].append(b)  # 정점 a에서 b로 이동
    # 진입차수를 1 증가
    indegree[b] += 1

# 위상 정렬 함수
def topology_sort():
    result = [] # 알고리즘 수행 결과를 담을 리스트
    q = deque()  # 큐 기능을 위한 deque 라이브러리 사용

    # 처음 시작할 때는 진입차수가 0인 노드를 큐에 삽입
    for i in range(1, v+1):
        if indegree[i] == 0:
            q.append(i)

    # 큐가 빌 때까지 반복
    while q:
        # 큐에서 원소 꺼내기
        now = q.popleft()
        result.append(now)
        # 해당 원소와 연결된 노드들의 진입차수에서 1 빼기
        for i in graph[now]:
            indegree[i] -= 1
            # 새롭게 진입차수가 0이 되는 노드를 큐에 삽입
            if indegree[i] == 0:
                q.append(i)

    # 위상 정렬을 수행한 결과 출력
    for i in result:
        print(i, end=' ')

topology_sort()
```

#### 시간 복잡도
위상 정렬의 시간 복잡도는 O(V + E)이다. <br>
위상 정렬을 수행할 때는 차례대로 모든 노드를 확인하면서, <br> 
해당 노드에서 출발하는 간선을 차례대로 제거해야 한다. <br>
결과적으로 노드와 간선을 모두 확인한다는 측면에서 O(V + E)의 시간이 소요되는 것이다.
