### 📌 개요
---
DFS / BFS와 최단 경로 알고리즘 모두 그래프 알고리즘의 한 유형으로 볼 수 있다. <br>
그래프 알고리즘은 굉장히 다양한데, 코딩 테스트에서 출제 비중이 낮은 편이지만, <br>
꼭 제대로 알아야 하는 알고리즘이다. <br>
여기서 다루는 개념들을 바르게 이해할 수만 있다면 다양한 응용문제들도 해결할 수 있게 된다. <br>


그래프 알고리즘은 앞서 배운 내용에 기반한다. <br>
예를 들어 크루스칼 알고리즘은 그리디 알고리즘으로 분류되며, <br>
위상 정렬 알고리즘은 큐 또는 스택 자료구조를 활용해야 구현할 수 있다. <br>


먼저 그래프란 노드와 노드 사이에 연결된 간선의 정보를 가지고 있는 자료구조를 말한다. <br>
예를 들어 '여러 개의 도시가 연결되어 있다'와 같은 내용이 등장하면 그래프 알고리즘을 의심해보면 된다. <br>


더불어 그래프 자료구조 중에서 <a style="color:red;">트리 자료구조</a>는 다양한 알고리즘에서 사용되므로 꼭 기억하자. <br>
트리 자료구조는 부모에서 자식으로 내려오는 계층적인 모델에 속한다. <br>
우선순위 큐를 구현하기 위해 최소 힙이나 최대 힙을 이용할 수 있는데, <br>
최소 힙은 항상 부모 노드가 자식 노드보다 크기가 작은 자료구조로서 트리 자료구조에 속한다고 할 수 있다. <br>

||그래프|트리|
|방향성|방향 그래프 혹은 무방향 그래프|방향그래프|
|순환성|순환 및 비순환|비순환|
|루트 노드 존재 여부|없음|있음|
|노드간 관계성|부모와 자식 관계 없음|부모와 자식 관계 있음|
|모델의 종류|네트워크 모델|계층 모델|

<br>

또한 그래프의 구현 방법은 2가지 방식이 존재한다. <br>

- <b>인접 행렬</b> : 2차원 배열을 사용하는 방식 <br>
노드의 개수가 V, 간선의 개수가 E라고 할 때, <br>
인접 행렬을 이용하는 방식은 O(V^2)만큼의 메모리 공간이 필요하며, <br>
두 노드간 간선의 비용을 O(1)의 시간으로 즉시 알 수 있다. <br>
인접 행렬은 다익스트라 알고리즘에 이용되는 방식이다. <br>

- <b>인접 리스트</b> : 리스트를 사용하는 방식 <br>
노드의 개수가 V, 간선의 개수가 E라고 할 때, <br>
인접 리스트를 이용할 때는 O(E)만큼의 메모리 공간이 필요하며, <br>
두 노드간 간선의 비용을 O(V)만큼의 시간을 소요하여 알 수 있다. <br>
인접 리스트는 플로이드 워셜 알고리즘에 이용되는 방식이다. <br>

노드와 간선의 개수가 모두 많으면 다익스트라 알고리즘, 노드의 개수가 적다면 플로이드 워셜 알고리즘을 이용할 수 있다. <br>

### ✅ 서로소 집합
<a style="color:red;">서로소 집합이란 공통 원소가 없는 두 집합을 의미한다.</a> <br>
<a style="color:magenta;">서로소 집합 자료구조란 서로소 부분 집합들로 나누어진 원소들의 데이터를 처리하기 위한 자료구조라고 할 수 있다.</a> <br>
서로소 집합 자료구조는 union과 find 2개의 연산으로 조작할 수 있다. <br>


union 연산은 2개의 원소가 포함된 집합을 하나의 집합으로 합치는 연산이다. <br>
find 연산은 특정한 원소가 속한 집합이 어떤 집합인지 알려주는 연산이다. <br>


서로소 집합 자료구조는 union-find 자료구조라고 불리기도 한다. <br>
두 집합이 서로소 관계인지를 확인할 수 있다는 말은 <br>
각 집합이 어떤 원소를 공통으로 가지고 있는지를 확인할 수 있다는 말과 같다.<br>

#### 서로소 집합 자료구조
서로소 집합 자료구조는 트리 자료구조를 이용하여 집합을 표현하는데, 알고리즘은 다음과 같다. <br>
    1️⃣ 노드의 개수(V) 크기의 부모 테이블을 모든 원소가 자기 자신을 부모로 가지도록 초기화한다. 
    2️⃣ union(합집합) 연산을 확인하여, 서로 연결된 두 노드 A, B를 확인한다.
      ◾ A와 B의 루트 노드 A', B'를 각각 찾는다.
      ◾ A'를 B'의 부모 노드로 설정한다(B'가 A'를 가리키도록 한다).
    3️⃣ 모든 union(합집합) 연산을 처리할 때까지 1번 과정을 반복한다.

##### 소스코드

```python
def find_parent(parent, x):
    if parent[x] != x:
    	return find_parent(parent, parent[x])
    return x

def union_parent(parent, a, b):
	a = find_parent(parent, a)
    b = find_parent(parent, b)
    if a<b:
    	parent[b] = a
    else:
    	parent[a] = b
        
v,e = map(int, input().split())
parent = [0]*(v+1)

for i in range(1, v+1):
	parent[i] = i
    
for i in range(e):
	a,b = map(int, input().split())
    union_parent(parent, a, b)
    
print('각 원소가 속한 집합: ', end='')
for i in range(1, v+1):
	print(find_parent(parent, i), end=' ')
    
print()

print('부모 테이블: ', end='')
for i in range(1, v+1):
	print(parent[i], end=' ')
```
