##### **정의**<br>

객체지향 분석이란, <br>
사용자의 요구사항을 분석하여 요구된 문제와 관련된 모든 클래스(객체), <br> 
이와 연관된 속성과 연산, 그들 간의 관계 등을 정의하여 모델링하는 작업을 의미 <br>


##### <br> **종류**
###### 럼바우 방법 : ```Rumbaugh```
  - 소프트웨어 구성 요소를 그래픽 표기법을 이용하여 <br>
  모델링하는 객체지향 분석(Object-oriented Analysis) 기법
  - 가장 일반적으로 사용하는 방법
  - 럼바우(Rumbaugh)의 분석 절차(객체모델링 → 동적 모델링 → 기능 모델링)
    - 객체 모델링
      - 객체 다이어그램(Object Diagram), 정보 모델링이라 표현
      - 시스템에서 요구하는 객체를 찾고 객체들 간의 관계를 정의
    - 동적 모델링
      - 상태 다이어그램(State Diagram)
      - 시간의 흐름에 따라 객체들 사이의 제어 흐름, 동작 순서 등의 동적인 행위
    - 기능 모델링
      - 자료 흐름도(Data Flow Diagram)
      - 프로세스들의 자료 흐름을 중심으로 처리 과정 표현

###### 부치 방법론 : ```Booch```
  - 미시적(Micro) 개발 프로세스와 거시적(Macro) 개발 프로세스 모두 포함하여 사용
  - Use-Case를 강조하여 활용한 방법
  - 클래스와 객체들을 분석 및 식별하고 클래스의 속성과 연산을 정의

###### 코드와 요든 방법론 : ```Code & Yourdon```
  - E-R 다이어그램을 사용하여 객체의 행위를 모델링
  - 객체 식별, 구조 식별

###### 워프스-브록 방법론 : ```Wirfs-Brock```
  - 분석과 설계 간 구분이 없으며, <br> 
  고객 명세서를 평가하여 설계 작업까지 연속적 수행
