### ✅ 정의
애플리케이션의 작동 상태, 이벤트 및 오류를 기록하는 중요한 개발 활동 <br>
주로 Log4j, Logback 또는 Java 표준 라이브러리로그를 사용

### ✅ 기본 단계

#### 1️⃣ Log4j 라이브러리 및 Gradle 의존성 설정
Gradle 프로젝트의 build.gradle 파일에 Log4j 라이브러리를 추가

```java
dependencies {
    implementation 'org.apache.logging.log4j:log4j-api:버전'
    implementation 'org.apache.logging.log4j:log4j-core:버전'
}
```

<br><br>

#### 2️⃣ Log4j2 설정 파일(선택사항)

```java
// log4j2.xml
<?xml version="1.0" encoding="UTF-8"?>
<Configuration status="INFO">
    <Appenders>
        <Console name="Console" target="SYSTEM_OUT">
            <PatternLayout pattern="%d{HH:mm:ss.SSS} [%t] %-5level %logger{36} - %msg%n"/>
        </Console>
    </Appenders>
    <Loggers>
        <Root level="info">
            <AppenderRef ref="Console"/>
        </Root>
    </Loggers>
</Configuration>
```

<br><br>

#### 3️⃣ Java 코드에서 로깅 사용

```java
public class LoggingExample {
    private static final Logger logger = LogManager.getLogger(LoggingExample.class);

    public static void main(String[] args) {
        logger.debug("Debug 메시지");
        logger.info("Info 메시지");
        logger.warn("경고 메시지");
        logger.error("에러 메시지");
    }
}
```

<br><br>

#### 4️⃣ 로그 출력 예제

```java
// Log4j2 설정을 따로 하지 않을 경우 로그 레벨이 INFO로 설정됨
16:29:44.123 [main] INFO  LoggingExample - Info 메시지
16:29:44.123 [main] WARN  LoggingExample - 경고 메시지
16:29:44.123 [main] ERROR LoggingExample - 에러 메시지
```

#### 5️⃣ 로그 레벨

- `**DEBUG**` 
DEBUG 레벨은 가장 낮은 로그 레벨이며, 디버깅 목적으로 사용됩니다. 애플리케이션의 내부 동작을 추적하고 디버그하는 데 사용됩니다. 일반적으로 개발 중에 사용하며, 프로그램이 어떻게 작동하는지 자세히 파악하고 문제를 해결하기 위해 로그를 기록하는 데 적합합니다.

- `**INFO**` 
INFO 레벨은 중요한 이벤트나 상태 변경을 나타내는 데 사용됩니다. 애플리케이션의 주요 이벤트나 상태 정보를 기록하는 데 사용됩니다. 예를 들어, 애플리케이션 시작 및 종료, 중요한 상태 전환 등을 로깅할 때 INFO 레벨을 사용합니다.

- `**WARN**` 
WARN (경고) 레벨은 특정 이벤트나 조건이 발생하였지만, 문제는 아직 아니라는 것을 나타냅니다. 예상치 못한 상황이 발생했지만 애플리케이션이 계속 작동할 수 있는 상황을 로깅할 때 사용됩니다. 이러한 로그는 잠재적인 문제를 알릴 때 유용합니다.

- `**ERROR**` 
ERROR (에러) 레벨은 심각한 문제나 예외 상황을 나타냅니다. 프로그램 실행에 중대한 오류가 발생한 경우, 에러 로그를 사용하여 이를 기록합니다. 이러한 로그는 시스템의 중단 또는 예외 처리를 통한 조치를 필요로 할 때 사용됩니다.
