### 📌 개요
---


<a style="color:red;">정렬이란 데이터를 특정한 기준에 따라서 순서대로 나열하는 것을 의미</a> <br>
데이터를 정렬하면 이진 탐색이 가능해진다. 즉 정렬 알고리즘은 이진 탐색의 전처리 과정이다.<br>
상황에 적절하지 못한 정렬 알고리즘을 사용하면 프로그램은 비효율적으로 동작하며, <br>
시간을 많이 소요하게 된다. <br>
그래서 정렬 알고리즘을 공부하다보면 자연스럽게 알고리즘 효율의 중요성을 깨닫는다. <br>
내림차순 정렬은 오름차순 정렬을 수행한 뒤에 그 결과를 뒤집기하여 내림차순 리스트를 만들 수 있다. <br>
리스트를 뒤집는 연산은 O(N)의 복잡도로 간단히 수행할 수 있으므로 오름차순을 위한 구현에 집중해도 된다.

<br><br>

### ✅ 선택 정렬
---


정렬 알고리즘 중 가장 원시적인 방법 <br>
가장 작은 데이터를 선택해 맨 앞에 있는 데이터와 바꾸고, <br>
그 다음 작은 데이터를 선택해 앞에서 두 번째 데이터와 바꾸는 과정을 반복한다. <br>
매번 가장 작은 것을 선택한다는 의미에서 선택 정렬 알고리즘이라고 한다.

#### 예제 소스 코드

```python
array = [7, 5, 9, 0, 3, 1, 6, 2, 4, 8]

for i in range(len(array)):
    min_index = i
    for j in range(i + 1, len(array)):
        if array[min_index] > array[j]:
            min_index = j
        array[i], array[min_index] = array[min_index], array[i]

print(array)    
```

<br>

#### 시간 복잡도

|데이터의 개수(N)|선택 정렬|퀵 정렬|기본 정렬 라이브러리|
|---|---|---|---|
|100|0.0123초|0.00156초|0.00000753초|
|1,000|0.354초|0.00343초|0.0000365초|
|10,000|15.475초|0.0312초|0.000248초|

선택 정렬은 O(N^2)의 시간 복잡도를 가지며 <br>
기본 정렬 라이브러리를 포함해 다른 정렬 알고리즘과 비교했을 때 매우 비효율적임 <br>
데이터가 정렬이 되어있지 않다면 더욱 비효율적으로 작동함 <br>
다만 특정한 리스트에서 가장 작은 데이터를 찾는 일이 잦으므로 익숙해질 필요가 있음

<br><br><br>

### ✅ 삽입 정렬
---


데이터를 하나씩 확인하여, 각 데이터를 적절한 위치에 삽입하는 방법 <br>
선택 정렬에 비해 실행 시간 측면에서 더 효율적인 알고리즘 <br>
선택 정렬은 현재 데이터의 상태와 상관없이 무조건 모든 원소를 비교하고 위치를 바꾸는 반면, <br>
삽입 정렬은 그렇지 않다. <br>
삽입 정렬은 특정한 데이터가 적절한 위치에 들어가기 이전에, <br>
그 앞까지의 데이터는 이미 정렬되어 있다고 가정한다. <br>
첫 번째 데이터는 정렬되어 있다고 판단하므로 두 번째 데이터부터 시작을 하게 된다.

#### 예제 소스 코드

```python
array = [7, 5, 9, 0, 3, 1, 6, 2, 4, 8]

for i in range(1, len(array)):
    for j in range(i, 0, -1):
        if array[j] < array[j - 1]:
            array[j], array[j - 1] = array[j - 1], array[j]
        else:
            break

print(array)    
```

<br>

#### 시간 복잡도

삽입정렬은 O(N^2)의 시간 복잡도를 가지지만 <br>
데이터가 거의 정렬되어 있는 상태인 최선의 경우 O(N)의 시간 복잡도를 갖는다. <br>
데이터의 정렬이 거의 되어 있는 상황에서는 퀵정렬 알고리즘보다 강력하다. <br>

<br><br><br>

### ✅ 퀵 정렬
---


퀵 정렬은 지금까지 배운 정렬 알고리즘 중에 가장 많이 사용되는 알고리즘이다. <br>
기준 데이터(피벗)를 설정하고 그 기준보다 큰 데이터와 작은 데이터의 위치를 바꾸는 알고리즘 <br>

    1️⃣ 리스트에서 첫 번째 데이터를 피벗으로 정한다.
    2️⃣ 왼쪽 끝에서부터 피벗보다 큰 데이터를 찾고, 오른쪽 끝에서부터 피벗보다 작은 데이터를 찾아 위치를 서로 교환한다.
    3️⃣ 왼쪽에서부터 찾는 값과 오른쪽에서부터 찾는 값의 위치가 서로 엇갈리는 경우에는 작은 데이터와 피벗의 위치를 서로 변경한다.
          엇갈리지 않았다면 작은 데이터와 큰 데이터의 위치를 서로 변경한다.
    4️⃣ 1 ~ 2번 과정을 반복한다.

퀵 정렬은 재귀 함수 형태로 구현했을 때 구현이 매우 간결해진다. <br>
또한 퀵 정렬이 끝나는 조건은 현재 리스트의 데이터 개수가 1개인 경우이다. <br>

#### 예제 소스 코드
```python
# 널리 사용되고 있는 직관적인 형태의 소스 코드
array = [5,7,9,0,3,1,6,2,4,8]

def quick_sort(array, start, end):
	if start >= end:
    	return
    pivot = start
    left = start + 1
    right = end
    while(left<=right):
        while(left<=end and array[left]<=array[pivot]):
        	left += 1
        while(right>start and array[tight]>=array[pivot]):
        	right -= 1
        if(left>right):
        	array[right], array[pivot] = array[pivot], array[right]
		else: 
        	array[left], array[right] = array[right], array[left]
    quick_sort(array, start, right-1)
    quick_sort(array, right+1, end)

quick_sort(array, 0, len(array)-1)
print(array)
```

```python
# 파이썬의 장점을 살린 소스 코드
array = [5,7,9,0,3,1,6,2,4,8]

def quick_sort(array):
    if len(array) <= 1:
    	return array
    pivot = array[0]  
    tail = array[1:]  
    
    left_side = [x for x in tail if x <= pivot] 
    right_side = [x for x in tail if x > pivot] 
    
    return quick_sort(left_side) + [pivot] + quick_sort(right_side)
    
print(quick_sort(array))
```

<br>

#### 시간 복잡도

퀵 정렬의 평균 시간 복잡도는 O(NlogN)이다. <br>
삽입 정렬과는 다르게 데이터가 거의 정렬되어 있는 상태에서 가장 비효율적으로 동작하는데, <br> 
최악의 경우 O(N^2)의 시간 복잡도를 갖는다. <br>
데이터의 정렬이 거의 되어 있는 상황에서는 퀵정렬 알고리즘보다 강력하다. <br>
퀵 정렬을 기반으로한 파이썬 기본 정렬 라이브러리는 <br>
추가적인 로직을 더하여 O(NlogN)을 보장해주므로 걱정하지 않아도 된다.

### ✅ 계수 정렬
---


계수 정렬은 데이터의 크기 범위가 제한되어 정수 형태로 표현할 수 있을 때 사용 가능하다. <br>
특정한 조건이 부합할 때만 사용할 수 있지만 매우 빠르게 동작하는 정렬 알고리즘이다. <br>
데이터의 개수가 N, 데이터(양수) 중 최댓값이 K일 때 최악의 경우에도 수행시간 O(N+K)를 보장한다. <br>
일반적으로 가장 큰 데이터와 가장 작은 데이터의 차이가 1,000,000을 넘지 않고, <br>
데이터가 많이 중복되어 있을수록 효과적이다. <br> 

    1️⃣ 가장 작은 데이터로부터 가장 큰 데이터까지의 범위가 모두 담길 수 있도록 리스트를 생성한다.
    2️⃣ 데이터를 하나씩 확인하며 데이터의 값과 동일한 인덱스의 데이터를 1씩 증가시킨다.
    3️⃣ 결과적으로 최종 리스트에는 각 데이터가 몇 번씩 등장했는지 그 횟수가 기록된다.
    4️⃣ 결과를 확인할 때는 리스트의 첫 번째 데이터부터 하나씩 그 값만큼 반복하여 인덱스를 출력한다.

#### 예제 소스 코드

```python
array = [7,5,9,0,3,1,6,2,9,1,4,8,0,5,2]

count = [0]*(max(array)+1)

for i in range(len(array)):
	count[array[i]] += 1  

for i in range(len(count)): 
	for j in range(count[i]):
    	print(i, end=' ') 
```

<br>

#### 시간 / 공간 복잡도

계수 정렬의 시간 / 공간 복잡도는 모두 O(N + K)이다.

<br><br><br>

### ✅ 파이썬의 정렬 라이브러리
---


알고리즘 문제를 풀 때는 앞서 다룬 예제처럼 정렬 알고리즘을 직접 작성하게 되는 경우도 있지만, <br>
미리 만들어진 라이브러리를 이용하는 것이 효과적인 경우가 더 많다. <br>
파이썬은 기본 정렬 라이브러리인 sorted() 함수를 제공한다.<br>
sorted() 함수는 병합 정렬을 기반으로 만들어졌다. <br>
병합 정렬은 일반적으로 퀵 정렬보다 느리지만 최악의 경우에도 시간 복잡도 O(NlogN)을 보장한다. <br> 
리스트 객체의 내장 함수인 sort()를 이용하는 방법도 있다.

<br>

#### 시간 복잡도

정렬 라이브러리는 최악의 경우에도 시간 복잡도 O(NlogN)을 보장한다. <br>
문제에서 별도의 요구가 없다면 단순히 정렬해야 하는 상황에서는 기본 정렬 라이브러리를 사용하고, <br>
데이터의 범위가 한정되어 있으며 더 빠르게 동작해야하는 경우에는 계수 정렬을 사용하자.

<br>

    3️⃣ 더 빠른 정렬이 필요한 문제 : 계수 정렬이나 기존에 알려진 알고리즘의 구조적인 개선을 거쳐야 함
