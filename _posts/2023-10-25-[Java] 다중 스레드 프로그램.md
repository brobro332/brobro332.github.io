### ✅ 정의
여러 스레드를 사용하여 병행성 문제를 다루고, 동시성 프로그램을 개발하는 방법

<br>
<hr style="border: 2px dashed #d3d3d3;">
<br>

### ✅ 기본 단계

#### 1️⃣ 스레드 생성 및 관리
자바에서 스레드를 생성하려면 Thread 클래스를 상속하거나 Runnable 인터페이스를 구현하여 스레드의 run 메서드를 정의 <br>
그런 다음 스레드 객체를 생성하고 start 메서드로 실행

```java
Thread myThread = new Thread();
myThread.start();
```

<br><br>

#### 2️⃣ 스레드 동기화 
병행 프로그래밍에서는 여러 스레드가 공유 데이터나 자원에 접근할 때 동기화가 필요할 수 있음 <br> 
synchronized 키워드나 java.util.concurrent 패키지의 동시성 클래스를 사용하여 스레드 간의 상호 배제를 구현 <br>
동기화 키워드 관해서는 더 자세히 다룰 예정

<br><br>

#### 3️⃣ 스레드 상태 및 라이프사이클
스레드는 여러 상태를 가짐 (NEW, RUNNABLE, BLOCKED, WAITING, TIMED_WAITING, TERMINATED) <br>
스레드의 라이프사이클을 이해하고, Thread 클래스의 메서드 및 스레드 상태 변화를 관찰하는 방법을 익혀야 함

<br><br>

#### 4️⃣ 스레드 간 통신
다중 스레드 프로그래밍에서 스레드 간의 통신은 중요한 부분 <br>
wait(), notify(), notifyAll()를 사용하여 스레드 간 메시지 전달 및 조건 변수를 활용하여 효율적인 통신을 구현

<br><br>

#### 5️⃣ Executor 프레임워크
java.util.concurrent 패키지의 Executor 프레임워크를 활용하여 스레드 풀을 관리하고 작업을 병렬로 실행 <br>
ThreadPoolExecutor 및 ForkJoinPool과 같은 클래스를 사용하여 스레드 풀을 커스터마이징할 수 있음

<br><br>

#### 6️⃣ 병렬 스트림 및 병렬 컬렉션:
Java 8 이상에서는 스트림 API를 사용하여 데이터 병렬 처리를 쉽게 수행 <br>
parallelStream() 및 java.util.concurrent 패키지의 병렬 컬렉션을 활용하여 작업을 병렬로 처리

<br><br>

#### 7️⃣ 동시성 문제 해결 및 교착상태 방지
데드락을 방지하기 위한 방법으로 tryLock(), synchronized 블록 순서화 등을 사용합니다.

### ✅ 예제 코드 및 출력 결과

```java
// 예제 코드
public class MultiThreadExample {
    public static void main(String[] args) {
        // 스레드 객체 생성
        MyThread thread1 = new MyThread("Thread 1");
        MyThread thread2 = new MyThread("Thread 2");

        // 스레드 시작
        thread1.start();
        thread2.start();

        // 메인 스레드에서도 작업을 수행할 수 있음
        for (int i = 0; i < 5; i++) {
            System.out.println("Main Thread: " + i);
        }
    }
}

class MyThread extends Thread {
    public MyThread(String name) {
        super(name);
    }

    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println(getName() + ": " + i);
        }
    }
}

// 출력 결과:  스레드 간의 실행 순서는 매번 다를 수 있음
Main Thread: 0
Thread 1: 0
Thread 2: 0
Main Thread: 1
Thread 1: 1
Thread 2: 1
Main Thread: 2
Thread 1: 2
Thread 2: 2
Main Thread: 3
Thread 1: 3
Thread 2: 3
Main Thread: 4
Thread 1: 4
Thread 2: 4
```
