### 📌 개요
---
컴퓨터는 연산 속도에 한계가 있고, <br> 
메모리 공간을 사용할 수 있는 데이터의 개수도 한정적이라는 점이 많은 제약을 발생시킨다. <br>
다만 다이나믹 프로그래밍 기법을 이용하면 메모리 공간을 약간 더 사용하여 연산 속도를 비약적으로 증가시킬 수 있다. <br>
다이나믹 프로그래밍으로 해결할 수 있는 대표적인 예시로 피보나치 수열이 있다. <br>

```python
# 재귀 함수로 구현한 피보나치 수열
def fibo(x):
    if x == 1 or x == 2:
        return 1
    return fibo(x - 1) + fibo(x - 2)

print(fibo(4))
```

그런데 피보나치 수열을 재귀 함수로 작성할 경우 단순히 매번 계산하도록 하면 <br>
n이 커질수록 수행 시간이 기하급수적으로 늘어 O(2^n)의 시간 복잡도를 갖게 된다. <br>
이는 동일한 함수가 반복적으로 호출되기 때문이다. <br>
이러한 문제는 다이나믹 프로그래밍을 사용하면 효율적으로 해결할 수 있다. <br>
다만 항상 다이나믹 프로그래밍을 사용할 수는 없으며 다음 조건을 만족해야만 한다. <br>
    1️⃣ 큰 문제를 작은 문제로 나눌 수 있다.
    2️⃣ 작은 문제에서 구한 정답은 그것을 포함하는 큰 문제에서도 동일하다.
다이나믹 프로그램이을 구현하는 방법 중 한 종류로, 한 번 구한 결과를 메모리 공간에 메모해두고 <br>
같은 식을 다시 호출하면 메모한 결과를 그대로 가져오는 기법인 메모이제이션 기법을 이용해보자. <br>

```python
# 재귀 함수 + 메모이제이션의 탑다운 방식으로 구현한 피보나치 수열
d = [0] * 100

def fibo(x):
    if x == 1 or x == 2:
        return 1
    if d[x] != 0:
        return d[x]
    d[x] = fibo(x - 1) + fibo(x - 2)
    return d[x]

print(fibo(99))
```

물론 재귀 함수를 사용하면 컴퓨터 시스템에서는 함수를 다시 호출했을 때 <br>
메모리 상에 적재되는 일련의 과정을 따라야 하므로 오버헤드가 발생할 수 있다. <br>
따라서 재귀 함수 대신에 반복문을 사용하는 것이 더 성능이 좋다. 

```python
# 반복문의 바텀업 방식으로 구현한 피보나치 수열
d = [0] * 100

d[1] = 1
d[2] = 2
n = 99

for i in range(3, n + 1):
    d[i] = d[i - 1] + d[i - 2]

print(d[n])
```

재귀 함수를 이용한 다이나믹 프로그래밍 기법은 큰 문제를 위해 작은 문제를 호출한다고 하여 탑다운 방식이라고 한다. <br>
반면 반복문을 이용하는 경우에는 작은 문제부터 차근차근 답을 도출한다고 하여 바텀업 방식이라고 말한다. <br>
다이나믹 프로그래밍의 전형적인 형태는 바텀업 방식이다. <br>
재귀 함수 깊이 관련 오류나 오버헤드 등의 문제를 고려하여 탑다운 보다는 바텀업 방식을 사용하는 것을 권장한다.
