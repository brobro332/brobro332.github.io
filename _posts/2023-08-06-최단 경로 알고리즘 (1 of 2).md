### 📌 개요
---
<a style="color:red;">최단 경로 알고리즘은 말 그대로 가장 짧은 경로를 찾는 알고리즘이다.</a> <br>
각 지점은 그래프에서 '노드'로 표현되고, 노드간 연결된 도로는 그래프에서 '간선'으로 표현된다. <br>
실제 코딩 테스트에서는 최단 경로를 모두 출력하는 문제보다는 단순히 최단 거리를 출력하도록 요구하는 문제가 많이 출제된다.<br>
최단 경로 알고리즘의 종류로는 다익스트라 최단 경로 알고리즘, 플로이드 워셜, 벨만 포드 알고리즘이 있다. <br>

<br><br>

### ✅ 다익스트라 최단 경로 알고리즘
<a style="color:red;">다익스트라 최단 경로 알고리즘은 그래프에서 여러 개의 노드가 있을 때, </a> <br>
특정한 노드에서 출발하여 다른 노드로 가는 각각의 최단 경로를 구해주는 알고리즘이며, GPS 소프트웨어의 기본 알고리즘으로 채택되곤 한다. <br>
다익스트라 알고리즘은 <b>그리디 알고리즘</b>으로 분류된다. <br>
매번 가장 적은 비용이 적은 노드를 선택해서 임의의 과정을 반복하기 때문이다. <br>

    1️⃣ 출발 노드를 설정한다.
    2️⃣ 최단 거리 테이블을 초기화한다.
    3️⃣ 방문하지 않는 노드 중에서 최단 거리가 가장 짧은 노드를 선택한다.
    4️⃣ 해당 노드를 거쳐 다른 노드로 가는 비용을 계산하여 최단 거리 테이블을 갱신한다.
    5️⃣ 위 과정에서 3 ~ 4번을 반복한다.

다익스트라 알고리즘은 최단 경로를 구하는 과정에서 <br>
'각 노드에 대한 현재까지의 최단 거리' 정보를 항상 1차원 리스트에 저장하며 리스트를 계속 갱신한다. <br>
위의 3 ~ 4번 과정을 수행한다는 점에서 그리디 알고리즘으로 분류될 수 있다. <br> 
다익스트라 알고리즘은 간단하게 구현하는 방법과, 개선된 방법 두 가지 종류로 나뉜다.

#### 간단한 다익스트라 알고리즘
간단한 다익스트라 알고리즘은 O(V^2)의 시간 복잡도를 가지며, 다익스트라에 의해 처음 고안된 알고리즘이다. <br>
V는 노드의 개수를 의미하며, <br>
방문하지 않은 노드 중에서 최단 거리가 가장 짧은 노드를 선택하기 위해 <br>
매 단계마다 1차원 리스트의 모든 원소를 순차 탐색한다. <br>

##### 소스코드

```python
# 간단한 다익스트라 알고리즘
import sys
input = sys.stdin.readline
INF = int(1e9)

n, m = map(int, input().split())
start = int(input())

graph = [[] for i in range(n + 1)]
visited = [False] * (n + 1)
distance = [INF] * (n + 1)

for _ in range(m):
    a, b, c = map(int, input().split())
    graph[a].append((b, c))

def get_smallest_node():
    min_value = INF
    index = 0
    for i in range(1, n + 1):
        if distance[i] < min_value and not visited[i]:
            min_value = distance[i]
            index = i
    return index

def dijkstra(start):
    distance[start] = 0
    visited[start] = True
    for j in graph[start]:
        distance[j[0]] = j[1]
    for i in range(n - 1):
        now = get_smallest_node()
        visited[now] = True
        for j in graph[now]:
            cost = distance[now] + j[1]
            if cost < distance[j[0]]:
                distance[j[0]] = cost

dijkstra(start)

for i in range(1, n + 1):
    if distance[i] == INF:
        print("INFINITY")
    else:
        print(distance[i])
```

##### 코드 해설
총 O(V)번에 걸쳐 최단 거리가 가장 짧은 노드를 매번 선형 탐색하고, <br>
현재 노드와 연결된 노드를 매번 일일이 확인하기 때문에 복잡도는 O(V^2)이다. <br>
최단 경로 문제에서 전체 노드의 개수가 5,000개 이하라면 이 코드로 문제를 풀 수 있지만, <br>
노드의 개수가 10,000개를 넘어가는 문제라면 개선된 다익스트라 알고리즘을 이용해야 한다. <br>

<br><br>

#### 개선된 다익스트라 알고리즘
개선된 다익스트라 알고리즘은 O(ElogV)의 시간 복잡도를 가지며, <br>
E는 간선의 개수를, V는 노드의 개수를 의미한다. <br>
개선된 다익스트라 알고리즘은 최단 거리가 가장 짧은 노드를 단순히 선형적으로 찾는 것이 아니라, <br>
<b>힙 자료구조</b>를 이용하여 가장 거리가 짧은 노드를 더욱 빠르게 찾을 수 있다. <br> 
이 과정에는 로그 시간이 걸린다. <br>
여기서 힙 자료구조는 우선순위 큐를 구현하기 위하여 사용하는 자료구조 중 하나이다. <br>
파이썬에서는 우선순위 큐가 필요할 때 PriorityQueue 또는 heapq를 사용할 수 있는데 <br>
heapq가 더 빠르게 동작하므로 일반적으로 이를 사용한다. <br>
우선순위 값을 표현할 때는 일반적으로 정수형 자료형의 변수가 사용된다. <br>
대부분의 프로그래밍 언어에서는 우선순위 큐 라이브러리에 (가치, 물건)과 같이 데이터의 묶음을 넣으면, <br>
첫 번째 원소인 '가치' 값을 기준으로 우선순위를 설정한다. <br>
이는 파이썬에서도 마찬가지이다. <br>
우선순위 큐를 구현할 때는 내부적으로 최소 힙 또는 최대 힙을 이용하는데, <br>
최소 힙의 경우 값이 낮은 데이터가 먼저 삭제되며 최대 힙은 반대로 값이 큰 데이터가 먼저 삭제된다. <br>  
다익스트라 최단 경로 알고리즘에서는 최소 힙 구조를 기반으로 하는 파이썬의 우선순위 큐 라이브러리를 그대로 사용하면 된다. <br>
최소 힙을 최대로 사용하기 위해 일부러 우선순위에 해당하는 값에 음수 부호를 붙여서 넣었다가, <br>
나중에 우선순위 큐에서 꺼낸 다음 다시 음수 부호를 붙여 원래의 값으로 되돌리는 방식을 사용할 수도 있다. <br>

|우선순위 큐 구현 방식|삽입 시간|삭제 시간|
|리스트|O(1)|O(N)|
|힙|O(logN)|O(logN)|

데이터의 개수가 N개일 때, <br>
힙 자료구조에 데이터를 모두 넣은 뒤 다시 모든 데이터를 꺼내는 경우 O(NlogN)의 시간 복잡도를 가지는 반면, <br>
리스트로 이를 구현할 경우 O(N^2)의 시간 복잡도를 갖게 된다. <br>
즉 우선순위 큐를 구현하는 데 리스트보다 힙 자료구조가 적합하다고 할 수 있다. <br>

##### 소스코드

```python
# 개선된 다익스트라 알고리즘
import heapq
import sys
input = sys.stdin.readline
INF = int(1e9)

n,m = map(int, input().split())
start = int(input())

graph = [[] for i in range(n + 1)]
distance = [INF] * (n+1)
for _ in range(m):
  a, b, c = map(int, input().split())
  graph[a].append((b,c))

def dijkstra(start):
  q = []

  heapq.heappush(q, (0, start))
  distance[start] = 0

  while q: 
    dist, now = heapq.heappop(q)
    if distance[now] < dist:
      continue
    for i in graph[now]:
      cost = dist + i[1]
      if cost < distance[i[0]]:
        distance[i[0]] = cost
        heapq.heappush(q, (cost, i[0]))

dijkstra(start)

for i in range(1, n+1):
  if distance[i] == INF:
    print("INFINITY")
  else:
    print(distance[i])
```

##### 코드 해설
최단 거리가 가장 짧은 노드를 선택하는 과정을 우선순위 큐를 이용하기 때문에, <br>
get_smallest_node() 함수를 작성할 필요가 없다는 특징이 있다. <br>
또한 한 번 처리된 노드는 더 이상 처리되지 않기 때문에 O(ElogV)의 시간 복잡도를 갖는다.<br>
노드의 개수가 10,000개를 넘어가는 문제라면 개선된 다익스트라 알고리즘을 이용해야 한다. <br>
다익스트라 최단 경로 알고리즘은 우선순위 큐를 이용한다는 점에서 <br> 
우선순위 큐를 필요로 하는 다른 문제 유형과도 흡사하다는 특징이 있다. <br>
예를 들어 그래프 문제로 유명한 최소 신장 트리 문제를 풀 때에도 <br>
Prim 알고리즘 등 일부 알고리즘의 구현이 흡사하다는 특징이 있다.
