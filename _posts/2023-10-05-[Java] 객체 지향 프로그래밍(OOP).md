### ✅ 클래스와 객체

#### 클래스
"객체의 설계도 또는 템플릿" <br>
객체를 생성하기 위한 구조와 동작을 정의 <br>
객체는 클래스의 인스턴스로, 클래스의 정의에 따라 생성된 실제 데이터와 메서드를 포함

<br>
<hr style="border: 2px dashed #d3d3d3;">
<br>

### ✅ 캡슐화(Encapsulation)
"데이터와 메서드를 하나로 묶음" <br>
캡슐화함으로써 데이터의 무결성을 보호하고, 외부에서 직접 접근하는 것을 제한

#### 접근 제한자
- public
모든 클래스에서 접근 가능한 가장 개방적인 접근 제한자
- protected
같은 패키지 내의 클래스와 해당 클래스를 상속한 하위 클래스에서 접근 가능
- default
접근 제한자를 지정하지 않았을 때의 기본값으로, 같은 패키지 내에서만 접근 가능
- private
해당 멤버에 대한 접근을 해당 클래스 내에서만 허용하며 외부 클래스에서는 접근할 수 없음

<br>
<hr style="border: 2px dashed #d3d3d3;">
<br>

### ✅ 상속(Inheritance)
"부모 클래스의 특성을 자식 클래스가 물려받음" <br>
코드 재사용성을 높이고 계층 구조를 형성하여 클래스 간의 관계 정의

#### extends
클래스가 다른 클래스를 상속하는 데 사용되는 키워드 <br>
Java에서는 단일 상속만 허용되므로 한 클래스는 하나의 부모 클래스만 상속받을 수 있음 <br>

```java
class Animal {
    void eat() {
        System.out.println("동물이 먹는다.");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("개가 짖는다.");
    }
}
```
Dog 클래스는 Animal 클래스를 상속하고 있으며, Dog 클래스는 eat 메서드를 사용할 수 있음

*implements는 상속과는 엄연히 다른 개념으로, 클래스가 인터페이스를 구현하기 위해 사용됨

<br>

### ✅ 다형성(Polymorphism)
"같은 이름의 메서드 또는 연산자가 다른 클래스에서 다른 방식으로 동작할 수 있는 능력" <br>
메서드 오버로딩과 오버라이딩을 통해 구현

#### 오버로딩
"같은 이름의 메서드를 여러 개 정의하는 것" <br>
같은 동작을 하는 메서드를 다양한 인자 형식으로 호출할 수 있음

#### 오버라이딩
"부모 클래스에 정의된 메서드를 자식 클래스에서 재정의하는 것"
